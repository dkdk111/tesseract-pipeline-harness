# Examples

## 01_market_brief

A complete, keyless run of the engine on one ordinary task: produce a competitive
market brief on the note-taking app space. It opens all four axes.

Files:

- `task.json`: the input. It declares the work's nature (iterative, a dependent
  sequence, independent branches, one oversized part), not its axes. The planner
  reads that nature and derives the four-axis structure.
- `task.md`: the same task described for a human reader.
- `tesseract.json`: the self-designed tree with the produced results. Generated by
  the engine, committed as a reference.
- `trace.md`: the human-readable self-design record. Generated, committed.
- `output.md`: the assembled brief the run produced. Generated, committed.

How the one task opened every axis:

- Time (outermost): the brief runs in revision rounds, not one pass.
- Order: inside a round, collect then analyze then write then review, a dependency
  chain where each step consumes the last.
- Breadth: inside collect, the competitors are gathered in parallel; inside analyze,
  the three lenses run in parallel.
- Depth: one competitor (Notion) is too large to summarize flat, so it becomes a
  seed and opens its own sub-structure.

## Run it yourself, no model and no keys

    # run the bundled example end to end and print the trace
    python -m tesseract_pipeline demo

    # or run this task explicitly and choose where the trace goes
    python -m tesseract_pipeline run examples/01_market_brief/task.json --out .tesseract/brief

    # or just render the committed trace
    python -m tesseract_pipeline render examples/01_market_brief/tesseract.json

The default worker is a deterministic simulator, so the run is fully reproducible
with nothing installed. The four axes, the sweep, and the box are real and really
executed; only the leaf content is synthetic.

## Regenerate the committed artifacts

    python -m tesseract_pipeline run examples/01_market_brief/task.json --out examples/01_market_brief

## Make the leaves real

To replace the simulator with a real model, see `../examples/llm_worker_example.py`.
Only the worker changes; the planner, the executor, and the box stay identical.
